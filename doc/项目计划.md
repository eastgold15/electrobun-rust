# Electrobun Rust 重构计划

## 目标

Electrobun 从 Zig→Rust 迁移 + CEF→wry 替换，目前已经完成大部分重构。

**最终架构：纯 Rust FFI + TS，零 C++/CEF** ✅ 已实现

```
TS (Bun) ──dlopen()──> electrobun_core.dll (Rust) ──wry/tao──> 原生窗口/WebView
```

---

## 关键决策

### IPC 设计：Eden 风格（方案A）

采用 **`#[eden_ipc]` 宏 + 代码生成** 实现端到端类型安全：

```rust
// Rust 侧定义 API
#[eden_ipc]
pub trait WindowAPI {
    fn create_window(&self, params: WindowParams) -> Result<Window, WindowError>;
}
```

宏自动生成：
1. FFI 导出函数（`#[no_mangle] extern "C" fn`）
2. JSON 序列化/反序列化
3. 全局实例管理
4. `.d.ts` 类型文件

### 开发方向：从上往下（TS → Rust）

```
Phase 1: TS API 设计            ✅
   ↓
Phase 2: #[eden_ipc] 宏实现     ✅
   ↓
Phase 3: 清理 CEF/C++           ✅
   ↓
Phase 4: Rust 实现 (tao/wry)    ✅ 基础版
   ↓
Phase 5: 重写 build.ts          ✅
   ↓
Phase 6: 集成测试               ⏳ 进行中
```

## 项目结构

```
electrobun/
├── crates/
│   ├── electrobun-macros/       — #[eden_ipc] / #[eden_stream] 宏
│   ├── electrobun-core/         — Rust FFI DLL (核心实现)
│   │   └── src/
│   │       ├── api/mod.rs       — API trait 定义 + ElectrobunApp 实现
│   │       ├── window.rs        — 窗口管理 (tao)
│   │       ├── webview.rs       — WebView (wry)
│   │       ├── tray.rs          — 系统托盘
│   │       ├── dialog.rs        — 对话框
│   │       ├── clipboard.rs     — 剪贴板
│   │       ├── shortcuts.rs     — 全局快捷键
│   │       └── ...
│   ├── electrobun-launcher/     — 启动器 (Bun 子进程管理)
│   ├── electrobun-extractor/    — 自解压程序
│   └── electrobun-build-tools/  — 构建工具 (asar, zstd)
├── packages/
│   └── electrobun-rust/
│       ├── cli/                 — CLI 工具 + build.ts
│       ├── core/                — TypeScript API 层
│       └── browser/             — 前端浏览器端代码
├── apps/
│   └── kitchen/                 — 示例应用
└── doc/
    └── 项目计划.md               — 本文件
```
