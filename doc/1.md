bun-win32 项目说明
项目概述
@bun-win32 是适配 Bun 运行时的 Win32 API 绑定工具集，内置12000+ Windows 原生接口，可让 Bun 直接调用全套 Win32 原生 API，彻底简化 Windows 平台下 Bun 应用的原生开发流程。
项目核心特性：零依赖、无需原生编译、完整类型适配，包含 112 个独立子包、12090 个可用函数、4500+ 类型定义。
核心技术优势
摒弃传统 Node 原生开发繁琐流程，无需 node-gyp、无需 C++ 编译器、无需预编译包。首次调用接口时自动解析系统符号，后续所有调用均直接使用缓存的原生指针，与直接调用 C 原生函数的执行路径完全一致。无数据编组层、无封装性能损耗，极致贴近原生性能。
该项目基于 Bun 纯 FFI（外部函数接口）能力实现，所有功能均为原生系统接口真实调用，无二次封装模拟逻辑。
核心功能能力
- 音频能力：基于 XAudio2 / DirectSound 实现完整的 PCM 音频合成与播放；支持 X3DAudio 三维空间音效计算，内置全部 XAPO 音频特效。
- 图形能力：支持 Direct3D 11/12、OpenGL（含 GLU）、Direct2D、DirectWrite 高清文字渲染（原生 ClearType 字形光栅化）、GDI+ 图像编码解码、WIC 多媒体编解码，以及 DirectComposition 实时窗口合成渲染，可通过 Bun 直接驱动 GPU 完成原生图形渲染。
- 安全与硬件能力：支持 Windows Hello 密钥认证流程、TPM 2.0 硬件加密指令、BitLocker 磁盘加密、AMSI 恶意代码检测；全程原生接口调用，无需调用 PowerShell 脚本中转。
- 系统底层能力：支持线程死锁实时检测、进程内 DISM/WIM 系统镜像处理、原生 NT 系统 API、故障转移集群 API、Windows 过滤平台（WFP）、ETW 事件跟踪解析等底层系统能力。
安装使用方式
支持按需安装，仅需引入项目所需功能子包，无需安装全部依赖：
bun add @bun-win32/d3d12 @bun-win32/xaudio2_9 @bun-win32/webauthn
运行环境要求
Windows 10 及以上系统、Bun 1.1.0 及以上版本。
配套开发文档
项目 112 个子包均内置 AI.md 说明文档，详细记录接口绑定规则与类型定义规范，可适配智能编码工具，保障开发调用精准无误。完整子包列表可查看官方仓库：https://github.com/ObscuritySRL/bun-win32

webview-bun 项目说明
项目简介
webview-bun 是面向 Bun 运行时的 WebView 桌面 GUI 库，基于轻量 WebView 内核实现，专门用于快速搭建**网页技术栈的桌面图形界面应用**。
发布信息：由开发者 Tr1cky（CA、管理员身份）于 2024 年 6 月 29 日发布 webview-bun v2.2.0 正式版本。本次更新重点释放了 Bun 原生编译能力，充分发挥 Bun 跨平台编译优势，补齐了桌面应用打包、跨端编译核心能力。
版本更新核心亮点
历经长期迭代、大量用户需求征集与社区讨论，v2.2.0 重磅上线，支持通过 Bun 原生命令直接打包桌面应用，无需额外编译工具链，大幅简化 Bun 桌面应用发布流程。
更新日志（Release notes）
- 单文件可执行程序打包：支持通过 bun build --compile 命令将 webview-bun 应用编译为**独立单文件可执行程序**。
- 跨平台编译：支持通过 --target 参数完成多平台交叉编译，一套代码可打包适配不同系统平台。
- 极致性能与体积优化：全新重构编译底层库，运行速度最大化优化，同时大幅缩减打包后文件体积。
- 移除后置下载脚本：所有依赖库文件已内置打包，彻底删除安装后自动下载脚本，完美适配跨平台编译场景，规避网络依赖与安装报错问题。
